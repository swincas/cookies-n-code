Introduction to HTML and CSS

The Basics: Tags, Elements, and Attributes

HTML stands for HyperText Markup Language, and is a markup language similar to LaTeX, Markdown, or RST. Initially developed in 1991, it's seen multiple versions over the years, with the latest major version being HTML 5 in 2014 (though it's up to 5.2 now). HTML 5 brought a lot of major improvements and removed some cruft from earlier versions, so it's kinda like the Python 3 of HTML except not as backwards-incompatible.

Tags

Everything in HTML lives inside tags. Tags are always enclosed in angle braces, like this: <...>. This document itself lives entirely inside the html tag. Most tags need to be closed, which is done by adding a / just after the opening brace. Here's an example of a properly-formatted tag:

<html>...</html>

For reasons, the html tag at the very beginning of your document needs to include the text "!DOCTYPE" at the start, like so: <!DOCTYPE html>.)

Within the html tag live two other important tags: the head and body tags. The head tag contains meta information about your webpage: things like the language it's in, or links to CSS stylesheets (we'll get to those!) or JavaScript files. The body tag contains the actual content of your webpage which will be visible to users.

(HTML is an incredibly forgiving language, in that browsers have all sorts of spaghetti code to deal with legacy features and improper formatting. For instance, it's often possible to get away without closing certain tags; browsers will make a best guess which is often correct. For the sake of everyone who will ever read your code [includin you], please do not abuse this.)

Document Structure

Within the body tag, you can have headers given within tags running from <h1> to <h6>, which give different sizes of headers. The p tag allows you to group text into paragraphs.

Two very useful tags to know about are the div and span tags. You can use these as convenient containers to hold whatever content you want. div is useful as a larger container holding multiple elements within it, while span is used more for inline test, like when you want to apply a particular style to a few words.

Another highly important tag is the a tag. This is what allows you to hyperlink to other webpages. (No, I don't know why a stands for "hyperlink".) To actually make a working hyperlink, we need to know about:

Attributes

Everything contained within a matching pair of tags is known as an element. Elements can have attributes, which are specified within the open tag. For example, to make a working hyperlink, you use the href attribute within an a tag, like so:

<a href="http://www.google.com">A link you can click to go to Google</a>

And here's what that would look like in action: A link you can click to go to Google.

There are a number of attributes in HTML, and different elements can have different attributes attached to them. To display an image, for instance, you'd use the src attribute within the image tag, like so:

<img src="some_image.png">

(Some tags, like img, don't require a matching end tag. Exceptions to rules make everyones' lives exciting! </sarcasm> [←not actually a real tag])

Some Important Attributes

Two very useful attributes that can be used in any element are the id and class attributes. id allows you specify a unique ID to any particular element, while class allows you to designate multiple elements as members of the same class. IDs and classes are completely up to you to decide, and allow you to easily specify styling to specific elements or sets of elements using CSS.

There's a lot more to talk about with HTML, but even with just this small collection of tags and attributes you can accomplish a lot. You can learn a lot more about it at the official W3C schools page: https://www.w3schools.com/html/default.asp.

CSS

CSS, which stands for Cascading Style Sheets, was introduced in the 1990s to solve a problem: HTML is intended to specify the layout and structure of a webpage, not the style. While browsers do have generic styles sheets which they apply to webpages (such as making an h1 element bigger than an h6 element), in general browsers apply minimal formatting to HTML, instead relying on an accompanying style sheet to dictate how a webpage should appear.

Rule-sets

CSS works by using a system of rule-sets, which select elements in the page using a variety of specification methods (called selectors). The rule-set then applies a particular style change to the element(s). The "cascading" part come from how these style changes propagate and override each other (which has a defined order, but is more advanced than needed here). This is also where the div and span tags and id and class attributes come into play.

A simple example: let's say I'd like to make particular words in my webpage be red. The way I've done it here is by using a span tag with a custom class attribute of "red-text", like this: <span class="red-text">red</span>. Next, I'll create a rule in my CSS file (we'll get to how to link the two in a second):

.red-text {
    color: red;
}

The period is used to indicate that this rule should apply to the class which is immediately following the period, in this case "red-text". If you instead want to apply a rule to all tags of a particular kind, you just put the tag name (without a period). If you want to make a rule to apply to a particular ID, you preface it with an octothorp (#) instead.

Applying CSS to a Webpage

Now, it's possible to apply CSS to a single element using the style attribute in that element, like this: <p style="color:red;">...</span>. This would give you a paragraph in which everything is red. It's not recommended, however, because the whole point of CSS is to separate structure from style, so putting it in your HTML document muddies the water and makes it harder to update.

Instead, the recommended way to use CSS is to put it in its own .css file which can be linked to from individual webpages. (Useful if you have a large website with many pages, all of which can link to one CSS file to get a consistent visual style.) To link a style sheet, use the link tag within the head of your document, with a couple of new attributes we haven't seen before (rel and type), and the href to provide the link to the file. Here's the element used in this document to link the spreadsheet in the same folder:

<link rel="stylesheet" type="text/css" href="example_stylesheet.css">

(link is another of those tags that don't need to be closed.)

By convention, the root webpage for a site is called index.html, and the directory it's in will be considered the root directory when specifying the path to the CSS file. So if your CSS file myStyle.css lives in the css directory inside the assets directory in the directory where your index.html is, the href attribute to link to it would be href="assets/css/myStyle.css".

CSS is incredibly powerful and versatile at this point, and this barely begins to scratch the surface. Like for HTML, I can recommend the official W3C webpage on CSS as a resource to find out what kinds of things you can do with it: https://www.w3schools.com/css/default.asp. CSS used to be limited to static styles, but you can now use it to do things like make simple animations and change the style of an element when people mouseover it. And if you want to get even more complicated, you can add JavaScript into your webpage to do even more things (but that's a whole 'nother tutorial on its own).

Now get out there and have some fun in the wonderful wide world of web design!
